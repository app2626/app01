# คู่มือติดตั้งและดูแลระบบ — Mobile Pre Order System (preorder3.2)

> อัปเดตล่าสุด: 2026-07-05 · สำหรับผู้ดูแลระบบ ไม่ต้องเขียนโค้ดเป็น

---

## 1. ระบบนี้คืออะไร

เว็บแอประบบจองสินค้ามือถือ ทำงานบน **Google Apps Script (GAS)** ใช้ **Google Sheets เป็นฐานข้อมูล** — ไม่มีค่า server รายเดือน ทุกอย่างอยู่ในบัญชี Google ของร้าน

ส่วนประกอบมี 2 อย่างเท่านั้น:

| ส่วน | อยู่ที่ไหน | ทำหน้าที่ |
|---|---|---|
| Google Sheets 1 ไฟล์ | Google Drive | ฐานข้อมูลทั้งหมด (สินค้า, ออเดอร์, สมาชิก, ตั้งค่า) |
| Apps Script (ผูกกับ Sheet) | เมนู Extensions → Apps Script | ตัวระบบ + หน้าเว็บ |

ซอร์สโค้ดมี 4 ไฟล์ใน `src/`: `Code.js` (หลังบ้าน), `Index.html` (โครงหน้าเว็บ), `JS.html` (หน้าบ้าน), `CSS.html` (สไตล์)

---

## 2. ติดตั้งครั้งแรก (ระบบใหม่ / ย้ายเครื่อง)

### 2.1 เตรียม Google Sheet + Apps Script
1. สร้าง Google Sheets เปล่า 1 ไฟล์ (ตั้งชื่อเช่น "MPOS Database")
2. เปิดเมนู **Extensions → Apps Script**
3. นำโค้ด 4 ไฟล์จาก `src/` ไปวาง: สร้างไฟล์ให้ชื่อตรงกัน (`Code.gs`, `Index.html`, `JS.html`, `CSS.html`)
   - ถ้าใช้ `clasp`: แก้ `scriptId` ใน `.clasp.json` ให้ชี้โปรเจกต์ใหม่ แล้วรัน `clasp push -f`
4. ใน Apps Script: **Project Settings** → เช็ค Time zone = `Asia/Bangkok`

### 2.2 Deploy เป็นเว็บแอป
1. ปุ่ม **Deploy → New deployment → Web app**
2. ตั้งค่า: Execute as = **Me** / Who has access = **Anyone** (หรือจำกัดตามองค์กร)
3. กด Deploy → อนุญาตสิทธิ์ (Authorize) → ได้ URL ลงท้าย `/exec` = **ลิงก์ใช้งานจริง**
4. URL ลงท้าย `/dev` (จาก Test deployments) = **ลิงก์ทดสอบ** เห็นโค้ดล่าสุดทันทีที่ push

### 2.3 สร้างฐานข้อมูล
1. เปิดลิงก์ระบบ → หน้า login มีปุ่ม **Auto Setup** → กด 1 ครั้ง (สร้างตารางครบทุกชีต + ข้อมูลตัวอย่าง)
2. เข้าสู่ระบบด้วยบัญชีเริ่มต้น:
   - Admin: `admin` / `admin123`
   - Sales ตัวอย่าง: `sales` / `sales123`
3. ⚠️ **เข้าได้แล้วให้เปลี่ยนรหัสทันที** ที่เมนู ตั้งค่าระบบ → Members (แก้ user admin ใส่รหัสใหม่)

### 2.4 ตั้งค่าก่อนเปิดใช้จริง (ตามลำดับ)
1. **เวลารับจอง** — ตั้งค่าระบบ → เวลารับจอง: กำหนด ReserveStart/End (นอกช่วงนี้ระบบไม่รับจอง ทั้งหน้าจอและหลังบ้าน)
2. **สินค้า** — Products: ใส่สินค้าจริง (ดูความหมายคอลัมน์ที่ข้อ 6.1) ลบตัวอย่างทิ้ง
3. **สาขา/ช่องทาง** — Branches, Channels ให้ตรงหน้างานจริง
4. **พนักงาน** — Members: สร้าง user จริง กำหนด Role (ดูข้อ 6.2) ลบ/เปลี่ยนรหัส user ตัวอย่าง
5. **ของแถม** — GiftMappings: ผูกรุ่นมือถือกับของแถมต่อช่องทาง
6. **โปรโมชัน** — Promotions (ส่วนลดท้ายบิล) และ Auto Promotions (ลด % เมื่อซื้อคู่)
7. **แบนเนอร์** — ตั้งค่าระบบ → Hero/Grid/Popup/พื้นหลัง login ใส่รูปของร้าน

## 3. การอัปเดตระบบ (เมื่อมีการแก้โค้ด)

ทำตามลำดับนี้**ทุกครั้ง**:

1. `clasp push -f` (หรือวางโค้ดใหม่ใน Apps Script editor แล้ว Save)
2. เปิดลิงก์ **`/dev`** → hard refresh (**Ctrl+Shift+R**) → ทดสอบ
3. ผ่านแล้ว: **Deploy → Manage deployments → เลือกตัว Web app → ✏️ → Version: New version → Deploy**
   - ⚠️ ข้ามขั้นนี้ = ลิงก์ `/exec` ที่พนักงานใช้จะยังเป็นโค้ดเก่า
4. ถ้าอัปเดตนั้นแก้เรื่อง token/secret: ผู้ใช้ทุกคนต้อง login ใหม่ 1 ครั้ง

---

## 4. เรื่องที่ต้องรู้เกี่ยวกับ Cache (สำคัญ — คนติดบ่อยที่สุด)

ระบบ cache ตารางที่อ่านบ่อย (Products, Promotions, Branches ฯลฯ) ไว้ **6 ชั่วโมง** เพื่อความเร็ว

- แก้ข้อมูล**ผ่านหน้าระบบ** → cache เคลียร์เอง เห็นผลทันที ✅
- แก้ข้อมูล**ในชีตโดยตรง** (พิมพ์ในเซลล์เอง / เพิ่มคอลัมน์) → หน้าเว็บอาจยังเห็นของเก่าได้ถึง 6 ชม. ⚠️

**วิธีเคลียร์ cache ทันที:** เข้าหน้า Products ในระบบ → เปิดสินค้าตัวไหนก็ได้ → กดบันทึก 1 ครั้ง (หรือทำแบบเดียวกันกับตารางที่แก้)

| งาน | ความถี่ | วิธี |

## 6. โครงสร้างข้อมูลที่ควรรู้

### 6.1 ชีต Products (คอลัมน์สำคัญ)
- `Model` — สินค้ารุ่นเดียวกันหลายความจุ/สี ใส่ Model เดียวกัน ระบบจะรวมเป็นการ์ดเดียวให้เลือกสี/ความจุ
- `Price` — ราคาขายจริง (ใช้คิดเงิน)
- `Original Price` — ราคาเดิมก่อนลด **ใส่เฉพาะรุ่นจัดโปร** → การ์ดขึ้นแถบ "ประหยัด ฿X" + ป้ายลด % อัตโนมัติ (เว้นว่าง = ไม่แสดง; ค่านี้ไม่กระทบราคาคิดเงิน)
- `Category` — `โมบาย` ถูกจำกัดจอง 1 เครื่อง/ออเดอร์โดยระบบ / `ของแถมแบรนด์`, `ของแถมช่องทาง` = สินค้าแถม
- `Status` — `เปิด`/`ปิด` (ปิด = ซ่อนจากหน้าจอง)
- ⚠️ ตัวเลขในชีตอย่าจัด format เป็นสกุลเงินที่มีสัญลักษณ์ ฿ (ระบบรองรับแล้วแต่ตัวเลขล้วนปลอดภัยสุด)

### 6.2 สิทธิ์ผู้ใช้ (Role)
- **Admin** — ทุกอย่าง
- **Manager** — ดูรายงาน/แก้สถานะออเดอร์ได้ ดูของแถม/Auto Promotions ได้ (อ่านอย่างเดียว) ลบออเดอร์ไม่ได้
- **Sales** — จองสินค้า + ดูออเดอร์เฉพาะสาขาตัวเอง
- ช่อง `Accessible Menus` ใน Members ใช้กำหนดเมนูรายคน (คั่นด้วย comma, `*` = ทั้งหมด)

### 6.3 กติกาห้ามทำในชีต (พังแล้วกู้ยาก)
- ❌ ห้ามลบ/สลับ/เปลี่ยนชื่อหัวคอลัมน์ที่มีอยู่ — ระบบอ้างอิงตามชื่อหัวคอลัมน์
- ❌ ห้ามแทรกคอลัมน์กลางตาราง Orders — คอลัมน์ใหม่ต้องต่อท้ายขวาสุดเท่านั้น
- ❌ ห้ามลบแถวใน Orders เอง — ให้ลบผ่านหน้าระบบ (ระบบจะคืนสต๊อก + ลง log ให้)
- ✅ ออเดอร์ 1 ใบ = หลายแถวในชีต (สินค้าหลัก + ของแถม + ส่วนลด) ผูกกันด้วย OrderID — ปกติ ไม่ใช่ข้อมูลซ้ำ

## 7. แก้ปัญหาที่พบบ่อย

| อาการ | สาเหตุ | วิธีแก้ |
|---|---|---|
| แก้สินค้าในชีตแล้วหน้าเว็บไม่เปลี่ยน | Cache 6 ชม. | บันทึกสินค้าผ่านหน้า Products 1 ครั้ง (ข้อ 4) |
| แถบ "ประหยัด" ไม่ขึ้น | ราคาเดิม ≤ ราคาขาย / หัวคอลัมน์สะกดไม่ตรง / cache | เช็ค `Original Price` สะกดเป๊ะ, ค่ามากกว่า Price, เคลียร์ cache |
| จองไม่ได้ ขึ้น "ไม่อยู่ในช่วงเวลาการจอง" | นอกช่วง ReserveStart/End | ตั้งค่าระบบ → เวลารับจอง |
| login ไม่ได้ ขึ้นให้รอ 5 นาที | ใส่รหัสผิดเกิน 5 ครั้ง (ระบบกันเดารหัส) | รอ 5 นาทีแล้วลองใหม่ |
| เบอร์โทรในชีตเหลือ 9 หลัก (0 หน้าหาย) | คอลัมน์ถูกเปลี่ยน format เป็นตัวเลข | เลือกคอลัมน์ → Format → Number → Plain text แล้วรัน `migrateDeobfuscatePII` ไม่ช่วยกรณีนี้ — ต้องแก้เบอร์ที่เสียเอง (ระบบเวอร์ชันปัจจุบันกันปัญหานี้แล้วสำหรับข้อมูลใหม่) |
| กดยืนยันจองแล้วขึ้น "การเชื่อมต่อใช้เวลานานเกินไป" | เน็ต/เซิร์ฟเวอร์ช้า | เช็คหน้า Orders ก่อนคีย์ซ้ำ — ถ้าคีย์ซ้ำโดยที่ใบแรกสำเร็จ ระบบจะจับซ้ำและไม่สร้างออเดอร์ใหม่ให้เอง |
| หน้าเว็บพังหลังอัปเดต | ยัง cache หน้าเก่า | Hard refresh (Ctrl+Shift+R) และเช็คว่าสร้าง New version แล้ว (ข้อ 3) |
